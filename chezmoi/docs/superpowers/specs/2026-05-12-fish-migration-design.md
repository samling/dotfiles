# Fish Migration Design

Date: 2026-05-12
Scope: side-by-side migration from zsh to fish. zsh files remain untouched. Login shell unchanged. User opts in by running `fish`.

## Goals

1. Port everything sensible from `dot_zsh/` and `dot_zshrc` to `dot_config/fish/`.
2. Drop zsh plugins that fish handles natively (syntax highlighting, autosuggestions, history-substring-search).
3. Replace fzf-tab + fzf zsh bindings with `PatrickF1/fzf.fish`.
4. Auto-install plugins on `chezmoi apply` via `run_onchange_` script.
5. Preserve local-dotfile sourcing pattern using `*.fish` glob in `~/dotfiles-private/fish/`, `~/work-dotfiles/fish/`, and `~/.config.fish.*`.

## Layout

```
dot_config/fish/
  config.fish                       # intentionally near-empty (conf.d/ auto-loaded)
  fish_plugins                      # fisher manifest
  conf.d/

    30-aliases.fish
```

Not committed (auto-managed by fish):
- `fish_variables` (universal variables, machine-local)
- `completions/` (auto-generated by tools)

## File-by-file mapping

### 00-path.fish ← dot_zsh/path.zsh

- Use `fish_add_path -gP <dir>` (idempotent, prepends, dedupes natively).
- Paths: `/usr/local`, `/usr/local/bin`, `/usr/local/sbin`, `/usr/bin`, `/bin`, `/usr/sbin`, `/sbin`, `~/.local/usr/bin`, `~/.local/bin`, asdf shims (`${ASDF_DATA_DIR:-$HOME/.asdf}/shims`), `/opt/homebrew/{bin,sbin}`, `~/.fzf/bin`.
- XDG_DATA_DIRS: `set -gx XDG_DATA_DIRS` with flatpak paths prepended.

### 10-env.fish ← dot_zsh/env.zsh

Direct ports via `set -gx`:
- Locales: `LC_ALL`, `LC_CTYPE`, `LANG`.
- `EDITOR=nvim`, `MANPAGER=...`, `RIPGREP_CONFIG_PATH`.
- `BROWSER` for WSL (guarded by `WSL_DISTRO_NAME`).
- FZF: `FZF_TMUX`, `FZF_TMUX_OPTS`, `FZF_ALT_C_OPTS`, `FZF_CTRL_R_OPTS`, `FZF_CTRL_T_OPTS`, `FZF_DEFAULT_OPTS` (with colors + flags), `FZF_DEFAULT_COMMAND`.
- GoLang: `unset GOROOT GOPATH`, PATH for go/bin/, conditional asdf golang.
- Krew, Kubernetes (KUBECONFIG built from `~/.kube/kubeconfigs/*.yaml *.yml`), Pulumi, Rust, Snap, KEYTIMEOUT, XAUTHORITY.
- Hyprland sig fix (guarded by `HYPRLAND_INSTANCE_SIGNATURE`).
- Zoxide `_ZO_DOCTOR=0`, `_ZO_FZF_OPTS`.
- `ARCHFLAGS="-arch (uname -m)"`.

### 20-options.fish ← dot_zsh/opt.zsh

- `set -g fish_greeting ""`.
- `set -g fish_history_max 100000` (replaces HISTSIZE/SAVEHIST/HISTFILESIZE).
- Most zsh setopts have no fish equivalent because fish behaves that way natively:
  - AUTO_CD: not in fish. Accept the loss; if it bites, add a postcmd handler later.
  - PUSHD: fish has `dirh`/`prevd`/`nextd`; skip.
  - HIST_*: fish dedupes and shares history natively.
  - GLOB_DOTS, CASE_GLOB: fish globbing differs; default behavior acceptable.

### 21-colors.fish

- Catppuccin Mocha palette via `set fish_color_*` (autosuggest, command, comment, error, param, quote, redirect, end, escape, operator, valid_path).
- Pair with pure prompt colors via `set pure_color_*`.

### 30-aliases.fish ← dot_zsh/aliases.zsh

Direct ports (all use `alias`):
- Builtin replacements: `cat`, `df` (duf), `diff` (delta), `journalctl`, `top`, `watch`, `vi`, `vim`.
- cd dots: fish loop `for i in (seq 2 25); alias (string repeat -n $i .) "cd "(string repeat -n (math $i - 1) ../); end`. Plus `alias .="cd ."`.
- Defaults: `ddi`, `df`, `rm`, `cp`, `mv`.
- Extras: `ai`, `cm`, `feh`, `glaze`, `jinaai`, `lg`, `piper`, `yless`.
- Hyprland: `hyprctl`, `hc`, `reload-waybar`, `reload-ags`, `debug-waybar`.
- Kubectl: `kubectl=kubecolor`, `k`, `kubectx`, `kx`, `kubens`, `kn`, `kb`.
- ls: `ls=lsd`, `l`, `ll`, `lll`.
- Shortcuts: `b64`, `open`, `o`, `v`, `R`, `Ra`, `which`.
- Tmux: `tmux="direnv exec / tmux"`.
- `nvault=vault`.

Dropped:
- `alias sudo="sudo "` (trailing-space trick) — no fish equivalent.
- `R=reload-zsh-configuration` → `R=reload-fish-configuration`.

### 40-keybindings.fish ← dot_zsh/keys.zsh

In `function fish_user_key_bindings`:
- `bind \cs cs-exec-widget` (Ctrl-S → command-snippets widget).
- `bind \cx\ce edit_command_buffer` (C-x C-e → fish builtin edit-in-editor).
- `bind \e\x7f backward-kill-word` (Alt-Backspace).
- `bind --erase \cl \cj \cp \cn` (free for tmux / disable defaults).
- `stty -ixon` in conf.d (frees Ctrl-S from terminal flow control).

Free from porting:
- UP/DOWN arrow history-substring-search: fish does prefix search natively.
- Ctrl-Right/Left forward/backward-word: fish defaults.
- Emacs mode: fish default.

### 50-tool-init.fish ← dot_zsh/completions.zsh + fnm.zsh

Sourced commands (guarded by `command -v`):
- `fnm env --use-on-cd --shell fish | source`
- `zoxide init fish --cmd cd | source`
- `direnv hook fish | source`
- `kubectl completion fish | source`
- `flux completion fish | source`
- `talosctl completion fish | source`
- `talhelper completion fish | source`
- `uv generate-shell-completion fish | source`
- `vkv completion fish | source`
- `zellij setup --generate-completion fish | source`
- `colima completion fish | source`
- `fx --comp fish | source`
- `vault` autocomplete (fish-aware via `complete -c vault -f -a '...'`).
- `complete -c kubecolor -w kubectl` (mirror kubectl completion).
- conda: `/home/sboynton/miniconda3/bin/conda shell.fish hook | source` if present.
- pure prompt: installed via fisher; no init call needed.

### 60-local-sources.fish

```fish
for f in ~/dotfiles-private/fish/*.fish ~/work-dotfiles/fish/*.fish ~/.config.fish.*
    test -r $f; and source $f
end
```

### 70-prompt.fish

- Reserved for pure prompt customization (`pure_color_*`, `pure_symbol_*`, `pure_threshold_command_duration`). Otherwise empty.

## Functions

Ported (`functions/<name>.fish`, one per file, autoloaded):
- `bd` — base64 decode `$argv[1]`.
- `be` — base64 encode `$argv[1]`.
- `wk` — viddy/watch wrapper for `kubectl $argv`.
- `wp` — viddy/watch the previous command (fish history via `history --max=2 | head -1`).
- `startssh` — parse `ssh-agent -s` (Bourne) output and convert to fish: `ssh-agent -s | string replace -r '^([A-Z_]+)=([^;]+);.*' 'set -gx $1 $2' | source`. Drops the trailing `echo Agent pid N;` line via `string match -v 'echo*'`.
- `cleanssh` — `pkill ssh-agent`; `set -e SSH_AGENT_PID SSH_AUTH_SOCK`; remove socket + pid file.
- `reload-fish` (alias target `R`/`Ra`) — exec fish in current pane; if `$TMUX` set and arg = `all`, broadcast `C-c` + `exec fish` to other tmux panes running fish.
- `cs-exec-widget` — `set -l cmd (cs exec); test -n "$cmd"; and commandline -i $cmd; commandline -f repaint`.

## Plugin install

`run_onchange_install-fish-plugins.sh.tmpl`:
```sh
#!/bin/sh
# fish_plugins hash: {{ include "dot_config/fish/fish_plugins" | sha256sum }}
set -e
if ! command -v fish >/dev/null; then
  echo "fish not installed; skipping"
  exit 0
fi
fish <<'EOF'
if not functions -q fisher
    curl -sL https://raw.githubusercontent.com/jorgebucaran/fisher/main/functions/fisher.fish | source
    fisher install jorgebucaran/fisher
end
fisher update
EOF
```

`fish_plugins`:
```
jorgebucaran/fisher
pure-fish/pure
PatrickF1/fzf.fish
```

The `# fish_plugins hash:` comment changes when `fish_plugins` changes, so chezmoi re-runs the script (run_onchange behavior).

## Deliberately not ported

- zsh-syntax-highlighting, zsh-autosuggestions, zsh-history-substring-search — fish native.
- fzf-tab — replaced by fzf.fish.
- OMZ git-auto-fetch — no direct fish equivalent. Drop.
- `prompt.zsh` — unused (zsh used pure prompt via completions.zsh).
- `zellij.zsh` tab-rename hook — mostly commented out.
- Functions not in user's list: reload-zsh, bw-unlock, git-status, git-recent, unset-aws, fman, jwt-decode, nvim-maybe-profile, rgv, s_debug, s, reset_terminal_colors, fzg.
- `alias sudo="sudo "` trailing-space alias-expansion trick.

## Out of scope

- Changing login shell (`chsh`).
- Deleting zsh files from chezmoi.
- Porting `dot_zsh/scripts/preview_all.sh` (still callable from fish FZF_* opts as-is).
